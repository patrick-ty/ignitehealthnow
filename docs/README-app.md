# Docs Guide

## Journals
- Journals live in `docs/journals/devjournal-XXX.md`.
- Files are sequentially numbered; pick the next available integer (e.g., if `devjournal-003.md` exists, the next is `devjournal-004.md`).
- Required structure:
  - `# Dev Journal XXX — <Concise Title>`
  - `## Context`
  - `## Decision(s)`
  - `## Rationale`
  - `## Alternatives Considered`
  - `## Constraints / Gotchas`
  - `## Follow-ups / TODOs`
- Start a new journal when the decision set/theme changes or when a new top-level section would be needed for a different decision area.
- Aim for ~400–800 words; keep to a single coherent change set (not a time log).
